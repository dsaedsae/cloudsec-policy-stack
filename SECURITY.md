# 보안 정책 (Security Policy)

이 저장소는 **교육·포트폴리오 레퍼런스**(학습 랩)입니다 — 프로덕션 배포물이 아닙니다. 그래도 보안 이슈 제보는 환영합니다.

## 범위
- 랩·스크립트·정책 예제의 **결함**(잘못된 통제, 우회 가능, 오해를 부르는 설명), 채점기 false-pass, 문서의 부정확한 보안 주장.
- 범위 밖: `THREAT_MODEL.md`에 "막지 못하는 것"으로 *명시된* 잔여 위험은 **알려진 한계**이지 취약점이 아닙니다.

## 제보 방법
- **민감하지 않은 것**: GitHub Issue로 (버그 템플릿 사용).
- **민감한 것**(악용 가능한 결함): GitHub **Security ▸ Report a vulnerability**(private advisory)를 이용하세요. 공개 Issue에 악용 코드를 올리지 마세요.
- 시크릿·키 노출을 발견하면 즉시 비공개로 알려주세요. (이 repo는 `gitleaks` CI로 시크릿 커밋을 차단합니다.)

## 응답
개인 포트폴리오라 공식 SLA는 없지만, 합리적 범위에서 확인·수정합니다. 수정은 `verify.sh`/단위테스트로 재검증한 뒤 반영합니다.

지원 브랜치: `master` (유일한 활성 브랜치).
